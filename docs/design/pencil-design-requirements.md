# Бриф для Pencil: дизайн MVP

## Общая задача

Сделать mobile-first дизайн приложения для заучивания и повторения шлок на санскрите. Интерфейс на русском. Первый экран после входа - рабочий дашборд, не лендинг.

Стиль: спокойный учебный инструмент для ежедневной практики. Без игрового давления, тяжелой религиозной стилизации, декоративных hero-блоков и лишних визуальных эффектов. Главное - читаемость, ясная навигация и быстрый ежедневный сценарий на телефоне.

Основной размер макетов: `390 x 844`. Дополнительно проверить `360 x 800`. Desktop-адаптацию подготовить позже, без отдельной desktop-only логики.

## Общие UI-правила

- Нижняя мобильная навигация: `Дашборд`, `Библиотека`, `Обучение`, `Настройки`.
- Использовать термины: `шлока`, `заучивание`, `повторение`, `серия дней`, `хочу выучить`, `на повторении`.
- Не использовать: `стих`, `тренировка`, `streak`, `аккаунт`.
- Карточка шлоки показывает название, источник/номер, статус и действие.
- Тексты должны помещаться на мобильном экране, длинные названия и слова должны переноситься без поломки верстки.
- Пустые состояния и ошибки нужно показать в макете, но без избыточной детализации.

## Этап 1: регистрация, вход, дашборд

Цель этапа: получить первые экраны, по которым можно согласовать базовый стиль, навигацию и главный ежедневный сценарий.

### 1. Регистрация

Нужен экран создания учетной записи.

Элементы:

- заголовок;
- поле email;
- поле пароль;
- кнопка регистрации;
- ссылка на вход;
- место для ошибки формы.

Не проектировать вход через Google, гостевой режим, подтверждение email и восстановление пароля.

### 2. Вход

Нужен экран входа в учетную запись.

Элементы:

- поле email;
- поле пароль;
- кнопка входа;
- ссылка на регистрацию;
- состояние ошибки неверных данных.

После успешного входа пользователь попадает на дашборд.

### 3. Дашборд

Главный экран ежедневной работы.

Обязательные блоки:

- верхний заголовок `Сегодня`;
- блок `Серия дней`;
- блок `Пачка повторения` с 1-5 шлоками;
- мягкий индикатор, что после пачки есть еще шлоки на сегодня, если они есть;
- список `Хочу выучить`, сначала 4-5 шлок;
- действие раскрыть весь список `Хочу выучить`;
- нижняя навигация из четырех разделов.

Нужно показать состояния:

- пустой дашборд нового пользователя с переходом в библиотеку;
- обычный дашборд со шлоками к повторению и заучиванию;
- все повторения на сегодня завершены;
- текущая пачка завершена, но можно взять следующую.

Важное ограничение: дашборд не должен давить количеством оставшихся повторений. Остаток показывать ненавязчиво.

### Примерные данные для этапа 1

- `Бхагавад-гита 1.1`
- `Бхагавад-гита 1.2`
- `Иша-упанишада 1`

Примеры CTA:

- `Повторить`
- `Учить`
- `В библиотеку`
- `Показать все`
- `Следующая пачка`

## Этап 2: остальные экраны

Цель этапа: после согласования этапа 1 подготовить полный набор экранов MVP.

### Библиотека шлок

Табы:

- `На повторении` - открыт по умолчанию;
- `Хочу выучить`;
- `Общая библиотека`.

Нужны поиск, карточки шлок, пустые состояния и состояние без результатов поиска.

### Страница шлоки

Показать:

- название;
- источник и номер;
- статус пользователя;
- пословную транслитерацию;
- пословный перевод;
- полный перевод, если есть;
- действие по статусу: `Добавить`, `Начать заучивание` или `Повторить`.

### Заучивание

Экран должен показывать шлоку, переводы и действия:

- `Выучил`;
- `Не выучил`;
- ссылка на советы в `Обучении`.

После `Выучил` нужен экран подтверждения с действиями:

- `Выучить еще`;
- `На дашборд`.

### Повторение

Нужны состояния потока:

1. Текст скрыт, пользователь пробует вспомнить шлоку.
2. Первая подсказка: половина первой строки.
3. Вторая подсказка: вся первая строка.
4. Полный текст.
5. Выбор результата после самостоятельного вспоминания:
   - `Все правильно`;
   - `Сделал ошибку`.

Также нужны действия:

- `Открыть подсказку`;
- `Показать весь текст`;
- переход к следующей шлоке или завершение потока.

### Обучение санскриту

Простой экран со списком раскрывающихся советов. Без курсов, квизов и прогресса.

Пример секций:

- `Как читать шлоку по строкам`;
- `Как учить через смысл слов`;
- `Что делать, если шлока не вспоминается`.

### Настройки

Показать:

- выбор типа транслитерации: `Кириллица` / `Латиница`;
- переключатель `Hard mode`;
- `Выйти из учетной записи`;
- вход в админку только для администратора.

### Админка шлок

Нужны базовые экраны:

- список шлок;
- создание шлоки;
- редактирование шлоки;
- публикация / скрытие.

Форма шлоки:

- код шлоки;
- источник;
- номер;
- отображаемое название;
- полный перевод, опционально;
- строки и слова;
- для каждого слова: кириллица, латиница, русский пословный перевод.

Не проектировать физическое удаление и пакетный импорт.

## Что отдать после каждого этапа

После этапа 1:

- mobile-макеты регистрации, входа и всех состояний дашборда;
- нижняя навигация;
- базовые компоненты: кнопки, поля, карточка шлоки, бейдж статуса, пустое состояние.

После этапа 2:

- mobile-макеты всех остальных экранов;
- состояния основных потоков;
- desktop-адаптация ключевых экранов;
- краткие правила дизайн-системы: цвета, типографика, кнопки, табы, карточки, статусы.
