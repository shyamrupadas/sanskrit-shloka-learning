# Использовать Evolution Design для архитектуры фронтенда

Для фронтенда используем Evolution Design как базовый подход к структуре приложения. ED поддерживает React, предлагает верхнеуровневые слои `app`, `features`, `services` и `shared`, а также делает акцент на архитектурных границах и эволюции модулей от простого к более структурированному по мере роста кода.

**Рассмотренные варианты**

- Начать без явной архитектурной формы и раскладывать код по мере появления экранов.
- Использовать Evolution Design с самого начала, но применять его прагматично, без преждевременного дробления модулей.

**Последствия**

Это задает будущим разработчикам и агентам понятные границы для страниц, фич, переиспользуемой бизнес-логики и shared-инфраструктуры. Цена решения — нужно следить, чтобы ED не превратился в лишнюю папочную церемонию: новые модули должны начинаться с простой структуры и усложняться только при реальной необходимости.
