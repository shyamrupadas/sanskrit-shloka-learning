# Использовать MongoDB как хранилище MVP

Для MVP используем MongoDB как основное хранилище данных. Структура шлоки естественно вложенная: строки, слова, варианты транслитерации и пословные переводы; история повторений также хорошо ложится на документную модель без необходимости сразу проектировать сложную реляционную схему.

**Рассмотренные варианты**

- Использовать реляционную базу данных и нормализовать шлоки, слова, переводы, личные статусы и историю повторений.
- Использовать MongoDB и хранить вложенные структуры шлок и связанные пользовательские данные в документной модели.

**Последствия**

MongoDB упрощает стартовую модель данных и админский ввод шлок, но требует аккуратно определить границы документов и индексы для пользовательского прогресса, алгоритма повторения и поиска по библиотеке.
