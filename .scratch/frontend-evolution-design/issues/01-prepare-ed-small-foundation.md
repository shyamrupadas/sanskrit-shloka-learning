# Подготовить ED small guardrails и shared-фундамент

Status: ready-for-human
Accepted: 2026-07-04

## Родитель

`.scratch/frontend-evolution-design/spec.md`

## Что сделать

Подготовить безопасный seam для последующих вертикальных срезов: включить автоматическую проверку ED small, создать живую спецификацию frontend-архитектуры, настроить временный migration mode и перенести низкорисковую глобальную инфраструктуру в `shared`. Существующие пользовательские сценарии после prefactoring должны работать без изменений.

Архитектурный workflow вынести в project skill, а scoped agent instructions оставить коротким маршрутизатором. Пока legacy-код остается в проекте, агент должен читать живую спецификацию перед любой frontend-задачей и не использовать legacy как образец.

## Критерии приемки

- [ ] Frontend lint запускается из общего project workflow.
- [ ] Frontend lint не зависит от build/generate tasks без необходимости и остается быстрой архитектурной проверкой.
- [ ] ESLint распознает `app`, отдельные модули `features` и `shared`.
- [ ] ESLint запрещает зависимости `shared → features/app`, `features → app` и обход public API фич.
- [ ] Public API фич допускает `index.ts(x)` и route-level `*.page.tsx`, но запрещает внешние импорты внутренних `ui`, `model`, `lib`, `api`.
- [ ] Архитектурные границы проверяются декларативным поддерживаемым ESLint plugin или явно обоснованным fallback-правилом.
- [ ] ESLint проверяет React hooks и предупреждает о рискованных fast-refresh exports без блокировки shadcn-style UI kit.
- [ ] Существующие UI-примитивы, i18n, общие utilities и API error helpers принадлежат подходящим группам `shared`.
- [ ] Генератор UI-примитивов создает новые элементы внутри shared UI kit.
- [ ] `docs/architecture/frontend.md` является источником текущих правил и отделяет постоянную целевую модель от временного migration mode.
- [ ] Корневые и scoped agent instructions направляют frontend-задачи к живой спецификации, запрещают расширять legacy и содержат обязательные команды проверки.
- [ ] Project skill ссылается на живую спецификацию без повторения правил, описывает аудит ED small, выбор этапа эволюции модуля и безопасную миграцию; skill проходит штатную валидацию.
- [ ] Строгий запрет неизвестных файлов и зависимостей явно отложен до финальной задачи, поскольку текущая legacy-структура еще не классифицирована как target architecture.
- [ ] ED small smoke-checks подтверждают запреты `shared → features/app`, `features → app`, обход public API и разрешенный импорт route-level `*.page.tsx`.
- [ ] Внешний вид, маршруты и пользовательское поведение не изменены.
- [ ] Lint, typecheck и unit-тесты проходят.
