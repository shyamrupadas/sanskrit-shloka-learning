# Подготовить ED small guardrails и shared-фундамент

Status: ready-for-agent

## Родитель

`.scratch/frontend-evolution-design/PRD.md`

## Что сделать

Подготовить безопасный seam для последующих вертикальных срезов: включить автоматическую проверку ED small, закрепить краткие повседневные правила и перенести низкорисковую глобальную инфраструктуру в `shared`. Существующие пользовательские сценарии после prefactoring должны работать без изменений.

Архитектурный workflow вынести в project skill, чтобы подробные правила загружались только для аудита и рефакторинга. Повседневные инструкции должны оставаться короткими и достаточными для правильного размещения нового frontend-кода.

## Критерии приемки

- [ ] Frontend lint запускается из общего project workflow.
- [ ] ESLint распознает `app`, отдельные модули `features` и `shared`.
- [ ] ESLint запрещает зависимости `shared → features/app`, `features → app` и обход public API фич.
- [ ] Существующие UI-примитивы, i18n, общие utilities и API error helpers принадлежат подходящим группам `shared`.
- [ ] Генератор UI-примитивов создает новые элементы внутри shared UI kit.
- [ ] Scoped agent instructions описывают слои, направления зависимостей, public API и обязательные команды проверки.
- [ ] Project skill описывает аудит ED small, выбор этапа эволюции модуля и безопасную миграцию; skill проходит штатную валидацию.
- [ ] Внешний вид, маршруты и пользовательское поведение не изменены.
- [ ] Lint, typecheck и unit-тесты проходят.
