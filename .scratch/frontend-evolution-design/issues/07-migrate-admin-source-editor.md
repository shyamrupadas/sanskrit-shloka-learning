# Перенести редактор Источника шлоки

Status: ready-for-agent

## Родитель

`.scratch/frontend-evolution-design/PRD.md`

## Что сделать

Добавить в `admin` полный подмодуль `source-editor`, который обслуживает создание и редактирование Источника шлоки через общую внутреннюю модель формы. Модель должна скрывать form state, преобразование структуры источника в request, immutable codes и validation.

Подмодуль использует общие внутренние admin UI-элементы, но не выносит admin-specific поля или сообщения в `shared`.

## Критерии приемки

- [ ] Routes создания и редактирования Источника шлоки открывают публичные страницы фичи `admin`.
- [ ] Создание и редактирование используют одну внутреннюю модель формы вместо дублирования mapping структуры.
- [ ] Поддерживаются структуры без глав, по главам и по частям с главами.
- [ ] Коды существующего Источника шлоки, частей и глав остаются неизменяемыми.
- [ ] Названия, описание, существующие разделы и добавление новых разделов редактируются как до рефакторинга.
- [ ] Source-editor tests покрывают create/edit, immutable fields, validation, success и API errors через публичные страницы.
- [ ] Legacy source editors удалены без compatibility re-export.
- [ ] Внешний вид, API-контракт и пользовательское поведение не изменены.
- [ ] Lint, typecheck и unit-тесты проходят.
