# Перенести Общую библиотеку шлок и страницу Шлоки

Status: ready-for-human
Accepted: 2026-07-04

## Родитель

`.scratch/frontend-evolution-design/PRD.md`

## Что сделать

Перенести полный library-сценарий в одну фичу: загрузка Общей библиотеки шлок, табы, Поиск шлок, изменение Статуса шлоки пользователя и открытие минимальной страницы Шлоки.

Внутри фичи отделить API state и пользовательские действия от отображения через небольшой интерфейс. Чистую логику поиска, фильтрации и выбора действия оставить тестируемой без React, не превращая каждый helper в отдельный модуль.

## Критерии приемки

- [ ] Library route и route конкретной Шлоки открывают публичные страницы одной фичи.
- [ ] Табы, Поиск шлок и empty states работают как до рефакторинга.
- [ ] Добавление Шлоки к заучиванию и удаление из списка сохраняют прежние API-вызовы и query invalidation.
- [ ] Карточка открывает минимальную страницу Шлоки, а возврат ведет в библиотеку.
- [ ] Чистые правила фильтрации и выбора card action тестируются через отдельный внутренний интерфейс.
- [ ] UI-тесты проверяют поведение через публичные страницы и общий test harness.
- [ ] Legacy library и shloka pages удалены без compatibility re-export.
- [ ] Внешний вид, route params, query keys и пользовательское поведение не изменены.
- [ ] Lint, typecheck и unit-тесты проходят.
