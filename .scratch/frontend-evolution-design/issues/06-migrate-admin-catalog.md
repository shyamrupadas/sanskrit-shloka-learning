# Перенести каталог Админки

Status: ready-for-agent

## Родитель

`.scratch/frontend-evolution-design/PRD.md`

## Что сделать

Создать фичу `admin` первым административным tracer bullet: защищенный каталог загружает Источники шлоки и вложенные Шлоки, отображает прежние подписи и предоставляет переходы к созданию и редактированию.

Каталог становится первым внутренним подмодулем `admin`. Его UI, query state и форматирование скрыты за route-level public API фичи. Редакторы Источника шлоки и Шлоки пока могут оставаться legacy routes, но не должны импортировать внутренности нового каталога.

## Критерии приемки

- [ ] Admin catalog route доступен только Администратору через существующий guard.
- [ ] Каталог принадлежит внутреннему подмодулю `catalog` фичи `admin`.
- [ ] App router импортирует каталог только через public API `admin`.
- [ ] Все Источники шлоки, включая пустые, отображаются как до рефакторинга.
- [ ] Подписи структуры, расположение Шлоки и excerpt сохраняют прежние правила.
- [ ] Переходы к созданию и редактированию сохраняют прежние routes.
- [ ] Catalog-тесты проверяют loading, error, пустые Источники шлоки и переходы через публичную страницу.
- [ ] Legacy catalog implementation удален без compatibility re-export.
- [ ] Внешний вид, API-вызов и пользовательское поведение не изменены.
- [ ] Lint, typecheck и unit-тесты проходят.
