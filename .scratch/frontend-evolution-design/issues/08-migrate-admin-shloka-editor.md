# Перенести редактор Шлоки

Status: ready-for-agent

## Родитель

`.scratch/frontend-evolution-design/PRD.md`

## Что сделать

Добавить в `admin` полный подмодуль `shloka-editor`, который обслуживает создание и редактирование Шлоки через общую внутреннюю модель формы. Модель должна скрывать выбор места в Структуре источника шлоки, нормализацию Пад шлоки, validation, immutable reference fields и преобразование request.

Подмодуль переиспользует внутренние admin UI-элементы, сформированные предыдущим срезом, но не расширяет public API фичи внутренними полями или helpers.

## Критерии приемки

- [ ] Routes создания и редактирования Шлоки открывают публичные страницы фичи `admin`.
- [ ] Создание и редактирование используют общую нормализацию Пад шлоки и validation.
- [ ] Выбор Источника шлоки, части и главы зависит от Структуры источника шлоки как до рефакторинга.
- [ ] Создание требует Ссылку на шлоку и заполненные Пады шлоки согласно текущим правилам.
- [ ] При редактировании Ссылка на шлоку и ее составляющие остаются неизменяемыми.
- [ ] Shloka-editor tests покрывают create/edit, выбор структуры, пустые Пады шлоки, immutable fields, success и API errors через публичные страницы.
- [ ] Legacy shloka editors и оставшийся admin monolith удалены без compatibility re-export.
- [ ] Внешний вид, API-контракт и пользовательское поведение не изменены.
- [ ] Lint, typecheck и unit-тесты проходят.
