# Администратор создает первую опубликованную шлоку

Status: ready-for-agent

## Родитель

`docs/prd/0001-initial-product.md`

## Что сделать

Дать администратору возможность создать валидную шлоку в общей библиотеке шлок и опубликовать ее, чтобы позже пользователи могли добавить ее в личный список. Этот срез должен заложить авторизацию администратора, защищенный административный маршрут шлок, правила валидации шлоки и форму хранения канонического содержимого шлоки.

## Критерии приемки

- [ ] Пользователь может иметь роль администратора, не теряя обычное пользовательское поведение и возможность иметь личный прогресс.
- [ ] Администраторы могут открыть защищенный маршрут управления шлоками; обычные пользователи не видят административную навигацию и не имеют доступа к административному API.
- [ ] Администратор может создать шлоку с кодом шлоки, источником шлоки, номером шлоки, отображаемым названием шлоки, опциональным полным переводом шлоки, строками и словами.
- [ ] Каждое слово требует пословную транслитерацию `ru`, пословную транслитерацию `en` и русский пословный перевод; деванагари не требуется.
- [ ] Код шлоки уникален и неизменяем после создания; пара источник шлоки + номер шлоки уникальна.
- [ ] Созданную шлоку можно опубликовать, после чего она появляется в API опубликованной общей библиотеки шлок в стабильном порядке по источнику и номеру.
- [ ] Тесты покрывают доступ администратора, отказ обычному пользователю, обязательные метаданные шлоки, обязательную структуру строк и слов, обязательные `ru` и `en` для каждого слова, обязательный русский пословный перевод, опциональный полный перевод, уникальный код, неизменяемый код, уникальную ссылку на шлоку и отсутствие обязательного деванагари.

## Заблокировано

Нет - можно начинать сразу
