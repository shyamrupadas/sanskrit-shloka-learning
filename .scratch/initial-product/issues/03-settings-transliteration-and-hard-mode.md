# Настройки типа транслитерации и hard mode

Status: ready-for-agent

## Родитель

`docs/prd/0001-initial-product.md`

## Что сделать

Добавить настройки учетной записи для типа транслитерации и hard mode. Экран настроек должен сохранять предпочтения пользователя, задавать тип транслитерации по умолчанию `ru` и отдавать сохраненные значения через API, чтобы экраны шлок могли показывать выбранную транслитерацию.

## Критерии приемки

- [ ] Экран настроек доступен авторизованным пользователям из нижней мобильной навигации.
- [ ] Пользователь может переключить тип транслитерации между кириллицей (`ru`) и латиницей (`en`), и выбор сохраняется после перезагрузки и между сессиями.
- [ ] Для новых учетных записей тип транслитерации по умолчанию равен `ru`.
- [ ] Пользователь может переключить hard mode, и выбор сохраняется после перезагрузки и между сессиями.
- [ ] Hard mode хранится как предпочтение, но не меняет поведение повторения в MVP.
- [ ] Экран настроек включает выход из учетной записи и показывает вход в админку только пользователям с ролью администратора.
- [ ] Тесты покрывают чтение и сохранение типа транслитерации, дефолт `ru`, чтение и сохранение hard mode, выход из настроек, видимость входа в админку только для администратора и отсутствие влияния hard mode на повторение в MVP.
