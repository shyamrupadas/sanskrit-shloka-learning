# Библиотека шлок и статус `буду учить`

Status: ready-for-agent

## Родитель

`docs/prd/0001-initial-product.md`

## Что сделать

Сделать пользовательскую библиотеку шлок с табами `Повторяю`, `Буду учить` и `Все`. Пользователь должен находить опубликованные шлоки, добавлять доступную шлоку в `буду учить`, убирать ее обратно в доступное состояние и видеть пустые состояния, объясняющие следующий шаг.

## Критерии приемки

- [ ] Маршрут библиотеки открывается с табом `Повторяю` по умолчанию.
- [ ] Таб `Все` показывает только опубликованные шлоки в стабильном порядке по источнику шлоки и номеру шлоки.
- [ ] Таб `Все` поддерживает поиск по отображаемому названию шлоки, источнику шлоки и номеру шлоки; бэкенд-поиск не обязателен, если фронтенд-поиска достаточно для MVP.
- [ ] Карточки шлок показывают отображаемое название шлоки, короткий фрагмент текста в выбранном типе транслитерации учетной записи, личный статус и действие для текущего таба.
- [ ] Пользователь может добавить доступную шлоку из `Все` в `буду учить`; после этого она появляется в `Буду учить` и больше не выглядит доступной для этого пользователя.
- [ ] Пользователь может убрать шлоку из `буду учить`; она возвращается в доступное состояние для этого пользователя.
- [ ] Есть пустые состояния для `Повторяю` и `Буду учить`, а также состояние без результатов поиска для `Все`.
- [ ] Тесты покрывают таб по умолчанию, только опубликованные шлоки в `Все`, сортировку по источнику и номеру, поиск, добавление в `буду учить`, удаление из `буду учить`, действия карточек по статусу, отображение выбранной транслитерации и обязательные пустые состояния.
