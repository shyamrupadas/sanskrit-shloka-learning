# 11 — Серия дней и значимая активность

**What to build:** Добавить расчет серии дней на основе локального дня пользователя и значимой активности. Значимая активность - завершенное повторение или перевод шлоки в `повторяю`; дашборд должен показывать ненулевой индикатор серии дней и состояние, продолжена ли серия сегодня.

**Blocked by:** None — can start immediately

**Status:** awaiting-human-review
Accepted: 2026-07-14

- [x] Бэкенд записывает значимую активность для завершенных повторений и для перевода шлоки в `повторяю`.
- [x] Серия дней считается по дню пользователя, а не по серверной дате.
- [x] Серия увеличивается не более одного раза за день пользователя после первой значимой активности.
- [x] Последовательные дни пользователя сохраняют и увеличивают серию; пропущенный день пользователя сбрасывает следующую значимую активность до серии 1.
- [x] Дашборд скрывает индикатор серии дней, когда серия равна нулю.
- [x] Ненулевой индикатор серии дней имеет ожидающее состояние до сегодняшней значимой активности и активное состояние после сегодняшней значимой активности.
- [x] Завершение повторения с дашборда или из библиотеки консистентно обновляет серию.
- [x] Тесты покрывают первую активность, несколько активностей в один день пользователя, последовательные дни, сброс после пропущенного дня, границы локального дня, состояние до/после сегодняшней активности, активность заучивания и активность повторения из обеих точек входа.

## Результат

- Завершенные повторения используют сохраненный `user_day`, а переход в
  `повторяю` теперь атомарно сохраняет `reviewing_started_user_day`,
  рассчитанный из переданной IANA-таймзоны и серверного времени события.
- Новый защищенный `GET /api/dashboard/streak` объединяет оба источника,
  считает уникальные последовательные календарные дни и возвращает число
  дней вместе с признаком продолжения серии сегодня.
- Дашборд скрывает нулевую серию, показывает приглушенное ожидающее состояние
  до первой активности текущего дня и цветное активное состояние после нее.
  Завершение заучивания и повторения инвалидирует dashboard queries.
- API unit tests защищают календарный расчет и оба persistence-пути; contract
  tests — таймзону и streak endpoint; web integration tests — состояния
  индикатора, заучивание и повторение с дашборда и из библиотеки.
- Добавлена миграция `0009_learning_activity_user_day`; ее необходимо применить
  в каждом окружении.
