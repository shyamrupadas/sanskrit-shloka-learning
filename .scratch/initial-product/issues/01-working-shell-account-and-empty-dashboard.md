# Рабочий скелет, учетная запись и пустой дашборд

Status: ready-for-human
Accepted: 2026-06-13

## Родитель

`.scratch/initial-product/spec.md`

## Что сделать

Сделать первый запускаемый сквозной срез продукта: монорепозиторий с API-границей от контракта, Nest-бэкендом, mobile-first React-оболочкой, русскими i18n-строками, регистрацией, входом, выходом, защищенными маршрутами и пустым дашбордом для нового авторизованного пользователя. Пустой дашборд должен быть первым экраном после входа и вести пользователя к добавлению первых шлок из библиотеки.

## Критерии приемки

- [x] В репозитории есть рабочий скелет приложения по принятым ADR: отдельные web и API приложения, TypeSpec как источник API-контракта, закоммиченные сгенерированные артефакты контракта и задокументированные базовые команды тестов и проверки типов.
- [x] Новый пользователь может зарегистрироваться по email, паролю и подтверждению пароля; пароли короче 6 символов отклоняются на фронтенде и бэкенде.
- [x] Существующий пользователь может войти по email и паролю, после входа видит дашборд и может выйти.
- [x] Формы регистрации и входа включают `Показать пароль`; на регистрации действие управляет паролем и подтверждением вместе, на входе - полем пароля.
- [x] Ошибка входа использует общее сообщение `Неверный email или пароль` и не раскрывает, существует ли email.
- [x] Защищенные маршруты приложения перенаправляют неавторизованных пользователей на экраны входа и регистрации.
- [x] Пустой дашборд авторизованного пользователя скрывает индикатор серии дней и блок повторения, показывает одно понятное действие `Добавить`, открывающее библиотеку, и использует мобильную оболочку с нижней навигацией.
- [x] Web-приложение готово к PWA на уровне MVP: есть манифест, иконки или заглушки для иконок, адаптивная mobile-first верстка, без зависимости от офлайн-прогресса.
- [x] Тесты покрывают валидацию регистрации, сообщение ошибки входа, поведение `Показать пароль`, вход, выход, защищенные маршруты и пустой дашборд.
