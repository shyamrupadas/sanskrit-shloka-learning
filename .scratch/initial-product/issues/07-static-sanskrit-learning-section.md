# Статический раздел `Обучение санскриту`

Status: ready-for-human
Accepted: 2026-07-12

## Родитель

`.scratch/initial-product/spec.md`

## Что сделать

Добавить раздел `Обучение санскриту` как статический список раскрывающихся статей-советов. В первой реализации можно использовать временный текст, написанный разработчиком; структура и навигация должны позволять позже заменить текст редакторски без добавления CMS.

## Критерии приемки

- [x] Раздел доступен из нижней мобильной навигации с короткой подписью `Обучение`.
- [x] Заголовок страницы использует краткое название `Советы` из Pencil-макета.
- [x] Страница показывает раскрывающиеся секции советов, например про чтение шлок по строкам, заучивание через смысл слов и действия, если шлока не вспоминается.
- [x] Временный текст статей написан на русском и не заявляет в UI или комментариях, что является финальным редакторским содержанием.
- [x] В разделе нет CMS, отслеживания прогресса, структуры курсов, квизов или административного редактирования.
- [x] Тесты покрывают переход в раздел, раскрытие/сворачивание советов, русские UI-строки через i18n и отсутствие курсов/квизов/прогресса.

## Принятые UI-решения

- Решение пользователя от 2026-07-12: заголовок страницы использует текст
  `Советы`, как на экране `Обучение санскриту` (`aj0kd`).
- Решение пользователя от 2026-07-12: одинаковые временные названия карточек
  сохраняются как в Pencil-макете; тексты статей раскрывают три темы, заданные в ticket.
- Решение пользователя от 2026-07-12: в нижней навигации активным является
  раздел `Обучение`, даже если текущий экранный экземпляр Pencil визуально
  сохраняет активное состояние `Дашборд`.

## Результат

- Добавлен защищенный маршрут `/learning` и доступный пункт `Обучение` в общей
  нижней навигации с корректным активным состоянием.
- Добавлен статический экран `Советы` с тремя русскими статьями из i18n и
  доступным раскрытием/сворачиванием через shared `TipAccordionItem`.
- Страница не делает API-запросов и не добавляет CMS, курсы, квизы, прогресс
  или административное редактирование.
- Route-level, component и browser-level тесты покрывают переход, защиту
  маршрута, активную навигацию, состояние советов и границы MVP.
- Backend, API-контракт и БД не менялись; новые DB migrations не требуются.
