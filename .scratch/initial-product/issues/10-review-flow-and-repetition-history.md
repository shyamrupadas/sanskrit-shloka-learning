# Поток повторения и история повторений

Status: ready-for-human
Accepted: 2026-07-13

## Родитель

`.scratch/initial-product/spec.md`

## Что сделать

Сделать поток повторения из дашборда и из карточек `Повторяю` в библиотеке. Поток должен проводить пользователя через вспоминание, подсказки, раскрытие полного текста, выбор результата и создание записей истории повторений только для завершенных повторений; незавершенные попытки не должны иметь побочных эффектов.

## Критерии приемки

- [x] Пользователь может начать повторение со шлоки к повторению сегодня на дашборде или через `Повторить` на карточке `Повторяю`.
- [x] Первый экран просит пользователя произнести шлоку по памяти до раскрытия текста.
- [x] Последовательность подсказок фиксирована: первая половина первой строки, затем вся первая строка, затем весь текст.
- [x] Открытие полного текста после неудачного вспоминания записывает результат `не смог вспомнить`.
- [x] Если пользователь вспомнил самостоятельно, приложение показывает полный текст и требует выбрать `Все правильно` или `Сделал ошибку` до завершения.
- [x] `Все правильно` записывает `вспомнил без ошибки`; `Сделал ошибку` записывает `вспомнил с ошибкой`; завершение после подсказки записывает `вспомнил с подсказкой`.
- [x] Завершенная запись истории повторений включает пользователя, шлоку, время завершения, день пользователя и результат.
- [x] Выход из потока повторения до завершения не создает запись истории и не убирает шлоку из сегодняшних кандидатов.
- [x] После завершения одной шлоки поток открывает следующую шлоку в текущем потоке повторения, если она есть; иначе возвращает к состоянию завершения на дашборде.
- [x] Тесты покрывают все четыре результата, вход с дашборда, вход из библиотеки, незавершенный выход без побочных эффектов, переход к следующей шлоке текущего потока и завершение дашборда после последнего кандидата.

## Принятые решения и design exceptions

- Пользователь разрешил design exception 2026-07-12: экземпляры кнопок экранов
  повторения получают функциональные подписи вместо унаследованных `Повторить`
  и `Добавить`; на второй подсказке добавляется недостающее действие.
- Поток использует действия `Вспомнил` / `Нужна подсказка`, затем
  `Вспомнил` / `Ещё подсказка` или `Показать весь текст`. После подсказки
  полный текст завершается результатом `вспомнил с подсказкой`; после
  самостоятельного вспоминания полный текст ведет к обязательному выбору
  `Все правильно` / `Сделал ошибку`.
- При входе фиксируется текущий авторитетный список кандидатов. После завершения
  поток идет по этому snapshot, а после последней шлоки возвращается на
  дашборд, который заново получает актуальное состояние.

## Результат

- Добавлен защищенный `POST /api/library/items/{shlokaCode}/complete-review`.
  Бэкенд принимает один из четырех результатов и IANA-таймзону, проверяет
  статус `повторяю` и сохраняет пользователя, шлоку, серверное время
  завершения, день пользователя и результат в существующую таблицу
  `shloka_reviews`.
- Добавлен route `/library/shlokas/$shlokaCode/review` с последовательными
  экранами скрытого текста, двух подсказок, полного текста и самостоятельной
  оценки результата. До завершения запрос записи не отправляется.
- Дашборд открывает поток по выбранной шлоке к повторению сегодня, а карточка
  `Повторяю` получила действие `Повторить`. После записи dashboard queries
  инвалидируются, поэтому завершенная шлока исчезает из кандидатов дня.
- API tests покрывают четыре результата, локальный день, обязательный статус
  и PostgreSQL-запись; contract test защищает публичную команду; web
  integration tests покрывают обе точки входа, подсказки, отсутствие побочных
  эффектов при выходе, переход по snapshot и завершение дашборда.
