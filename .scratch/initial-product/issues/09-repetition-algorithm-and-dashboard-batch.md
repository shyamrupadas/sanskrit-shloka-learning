# Алгоритм повторения и пачка повторения на дашборде

Status: ready-for-human
Accepted: 2026-07-12

## Родитель

`.scratch/initial-product/PRD.md`

## Что сделать

Добавить алгоритм повторения на бэкенде и подключить его к дашборду. Дашборд должен показывать начальную пачку повторения, компактное раскрытие `Показать еще N` для оставшихся шлок к повторению сегодня, компактный список шлок к заучиванию и нужные пустые/завершенные состояния без объяснения пользователю причин алгоритма.

## Критерии приемки

- [x] Алгоритм повторения выбирает кандидатов только из шлок в статусе `повторяю`.
- [x] Кандидаты исключают шлоки, уже повторенные в текущий день пользователя, и шлоки, переведенные в `повторяю` в текущий день пользователя.
- [x] Порядок приоритетов кандидатов соответствует PRD: последний результат `не смог вспомнить`, последний результат `вспомнил с ошибкой`, последний результат `вспомнил с подсказкой`, шлока переведена в `повторяю` менее 5 дней назад, шлока не повторялась более 5 дней, затем остальные шлоки в `повторяю`.
- [x] Внутри одного приоритета раньше показываются шлоки, которые дольше не повторялись; для шлок без истории повторений используется время перехода в `повторяю`.
- [x] Отдельный endpoint шлок к повторению принимает задаваемый фронтендом лимит; начальный запрос с лимитом 5 возвращает до 5 шлок и количество остальных кандидатов.
- [x] `Показать еще N` запрашивает с отдельного endpoint свежий полный список кандидатов, не повторенных сегодня; полный ответ является авторитетным, включает уже показанные актуальные шлоки и позволяет добавить новые снизу без перемонтирования неизменившихся карточек.
- [x] Отдельный endpoint шлок к заучиванию принимает задаваемый фронтендом лимит; начальный запрос с лимитом 3 возвращает до 3 шлок и количество остальных, а `Показать все` запрашивает и раскрывает полный список на месте.
- [x] Дашборд включает пустые состояния для списков повторения и заучивания, а также встроенное состояние завершения, когда все сегодняшние кандидаты выполнены.
- [x] Тесты покрывают исключения алгоритма, все уровни приоритета, сортировку внутри приоритета, начальный лимит 5, количество оставшихся, `Показать еще N`, влияние ручного повторения на количество, раскрытие списка заучивания на дашборде и отсутствие пользовательского объяснения алгоритма.

## Принятые решения и design exceptions

- Design exception по решению пользователя от 2026-07-12: раскрытые состояния
  после `Показать еще N` и `Показать все` реализуются без отдельных Pencil-макетов.
- Для списков используются отдельные backend endpoints. Фронтенд передает
  лимит 5 для начальной пачки повторения и лимит 3 для начального списка
  заучивания; запрос без лимита возвращает соответствующий полный список.
- Полный ответ является авторитетным и повторно содержит начальный набор.
  Фронтенд сверяет элементы по коду: неизменившиеся карточки сохраняются,
  новые добавляются снизу, устаревшие удаляются.
- Любое завершенное повторение, включая результат `не смог вспомнить`, убирает
  шлоку из кандидатов текущего дня пользователя. Незавершенный выход не влияет
  на список.
- Счетчик остальных элементов является снимком. Полный запрос может вернуть
  ноль новых элементов после параллельного изменения; отдельное UI-состояние
  для этого не добавляется.
- Во время раскрытия текущий список остается видимым, действие временно
  недоступно. При ошибке список сохраняется, действие снова доступно, отдельное
  сообщение не добавляется.
- Порядок детерминирован: повторение использует приоритет алгоритма,
  эффективное время последнего повторения и код шлоки; заучивание использует
  время добавления в `буду учить` и код шлоки.
- Фронтенд передает IANA-таймзону браузера, бэкенд валидирует ее и рассчитывает
  текущий день пользователя.
- После успешного раскрытия действие исчезает без анимации, но его место
  сохраняет отступ. Внутреннего скролла нет; прокручивается весь дашборд.
  Плотность и отступы карточек не меняются.

## Результат

- Добавлены защищенные endpoints `/api/dashboard/review-shlokas` и
  `/api/dashboard/learning-shlokas`: положительный `limit` задает начальный
  срез, отсутствие лимита возвращает полный авторитетный список.
- Backend вычисляет день пользователя из валидированной IANA-таймзоны,
  исключает завершенные и только что переведенные в повторение шлоки,
  применяет все приоритеты и детерминированную сортировку.
- Добавлена таблица `shloka_reviews` для завершенной истории повторений и
  индексы для текущего дня и последнего результата. Миграцию
  `0007_review_history` необходимо применить в каждом окружении.
- Дашборд показывает до 5 шлок к повторению и до 3 к заучиванию, раскрывает
  свежие полные списки без анимации и внутреннего скролла, сохраняет карточки
  с теми же кодами и поддерживает пустые/завершенное состояния.
- Выбор карточки в блоке `Выучите шлоки` открывает focused-flow заучивания
  `/library/shlokas/$shlokaCode/learn`, а не страницу просмотра шлоки.
- API tests защищают алгоритм и validation, contract test — два endpoint и
  query-параметры, web integration tests — пользовательские состояния и оба
  раскрытия. Browser smoke/E2E подтверждает мобильную композицию и навигацию.
