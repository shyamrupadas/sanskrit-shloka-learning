# Редактирование и скрытие шлок без сброса прогресса

Status: ready-for-agent

## Родитель

`docs/prd/0001-initial-product.md`

## Что сделать

Завершить административное управление шлоками: редактирование, публикацию и скрытие без физического удаления и сброса прогресса. Правки канонической шлоки должны обновлять текст для всех пользователей, скрытые шлоки должны исчезать из `буду учить`, а скрытые шлоки, уже находящиеся в `повторяю`, должны оставаться в повторении.

## Критерии приемки

- [ ] Администраторы могут редактировать существующие метаданные шлоки, канонический текст, пословные транслитерации, пословные переводы, опциональный полный перевод и статус публикации.
- [ ] Редактирование уже опубликованной шлоки показывает предупреждение, что изменение затронет всех пользователей.
- [ ] Редактирование канонического содержимого шлоки обновляет отображение для всех пользователей без сброса личного статуса шлоки или истории повторений.
- [ ] В административном UI и API MVP нет действия физического удаления.
- [ ] Скрытие шлоки не позволяет новым пользователям добавить ее из `Все`.
- [ ] Скрытие шлоки удаляет ее из списков `буду учить` пользователей.
- [ ] Скрытая шлока, уже находящаяся в `повторяю`, остается в `повторяю`, остается видимой пользователю там, где это нужно, и продолжает участвовать в алгоритме повторения.
- [ ] Тесты покрывают предупреждение при редактировании опубликованной шлоки, отсутствие физического удаления, отсутствие сброса прогресса после правок, исчезновение скрытой шлоки из `буду учить`, недоступность скрытой шлоки для нового добавления, сохранение скрытой выученной шлоки в `повторяю` и участие скрытой выученной шлоки в повторении.
