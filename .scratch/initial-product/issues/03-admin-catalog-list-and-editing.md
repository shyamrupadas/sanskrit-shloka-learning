# Админка со списком шлок и редактированием каталога

Status: ready-for-agent

## Родитель

`docs/prd/0001-initial-product.md`

## Что сделать

Добавить защищенную страницу `Админка` для администратора: один входной экран административного раздела со ссылками `Новая шлока`, `Новый источник` и списком шлок, сгруппированным по источникам шлок. Страница не добавляется в нижнюю навигацию; переход из настроек появится в следующем срезе настроек. Этот срез также завершает административное редактирование каталога: источник шлоки и шлока открываются из списка через защищенные маршруты редактирования.

Для списка нужна отдельная защищенная admin-ручка чтения каталога. Она может переиспользовать тот же catalog layer, что и пользовательская библиотека, но должна возвращать все источники шлок, включая источники без шлок. Пользовательская библиотека шлок не должна получать пустые источники.

## Критерии приемки

- [ ] Маршрут `Админка` доступен только пользователям с ролью администратора; обычные пользователи не имеют доступа по прямому URL/API-запросу.
- [ ] `Админка` не появляется в нижней навигации приложения.
- [ ] Верхние действия `Новая шлока` и `Новый источник` открывают защищенные формы создания из предыдущего среза.
- [ ] `Список шлок` показывает все источники шлок, включая источники без шлок.
- [ ] Для пустого источника не показывается отдельный плейсхолдер и не предлагается добавить шлоку.
- [ ] Подпись источника выводится из структуры источника: `N глав` для источника с главами, `N частей` для источника с частями и `N шлок` для источника без глав.
- [ ] Шлоки внутри источника показаны в стабильном порядке по структуре источника и номеру шлоки.
- [ ] Строка шлоки в админке показывает номер или место шлоки и короткий фрагмент канонического текста кириллицей, не дублируя название источника в каждой строке.
- [ ] Действие редактирования на источнике открывает защищенную форму редактирования источника.
- [ ] Форма редактирования источника позволяет менять название и описание источника, названия существующих частей и глав, а также добавлять новые части и главы.
- [ ] Форма редактирования источника не позволяет менять код источника, тип структуры источника, коды существующих частей и глав или удалять существующие части и главы.
- [ ] Действие редактирования на шлоке открывает защищенную форму редактирования шлоки.
- [ ] Форма редактирования шлоки показывает источник, часть, главу и поле `Текст` как неизменяемую ссылку на шлоку.
- [ ] Форма редактирования шлоки позволяет менять только пада и опциональный полный перевод шлоки; первая пада остается обязательной.
- [ ] Редактирование шлоки показывает предупреждение, что изменение канонического текста затронет всех пользователей общей библиотеки.
- [ ] Код источника, код шлоки и ссылка на шлоку остаются неизменяемыми после создания.
- [ ] В административном UI и API текущего MVP нет физического удаления, публикации, скрытия или пакетного импорта.
- [ ] Тесты покрывают доступ администратора, отказ обычному пользователю, защищенную admin-ручку списка, отображение всех источников включая пустые, отсутствие плейсхолдера для пустого источника, стабильную сортировку шлок, переходы к созданию и редактированию, ограничения редактирования источника, ограничения редактирования шлоки, обязательную первую паду при редактировании и отсутствие удаления/публикации/скрытия.
