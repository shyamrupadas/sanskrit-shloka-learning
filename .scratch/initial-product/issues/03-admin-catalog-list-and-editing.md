# Админка со списком шлок и редактированием каталога

Status: ready-for-human
Accepted: 2026-06-27

## Родитель

`.scratch/initial-product/spec.md`

## Что сделать

Добавить защищенную страницу `Админка` для администратора: один входной экран административного раздела со ссылками `Новая шлока`, `Новый источник` и списком шлок, сгруппированным по источникам шлок. Страница не добавляется в нижнюю навигацию; переход из настроек появится в следующем срезе настроек. Этот срез также завершает административное редактирование каталога: источник шлоки и шлока открываются из списка через защищенные маршруты редактирования.

Для списка нужна отдельная защищенная admin-ручка чтения каталога. Она может переиспользовать тот же catalog layer, что и пользовательская библиотека, но должна возвращать все источники шлок, включая источники без шлок. Пользовательская библиотека шлок не должна получать пустые источники.

## Критерии приемки

- [x] Маршрут `Админка` доступен только пользователям с ролью администратора; обычные пользователи не имеют доступа по прямому URL/API-запросу.
- [x] `Админка` не появляется в нижней навигации приложения.
- [x] Верхние действия `Новая шлока` и `Новый источник` открывают защищенные формы создания из предыдущего среза.
- [x] `Список шлок` показывает все источники шлок, включая источники без шлок.
- [x] Для пустого источника не показывается отдельный плейсхолдер и не предлагается добавить шлоку.
- [x] Подпись источника выводится из структуры источника: `N глав` для источника с главами, `N частей` для источника с частями и `N шлок` для источника без глав.
- [x] Шлоки внутри источника показаны в стабильном порядке по структуре источника и номеру шлоки.
- [x] Строка шлоки в админке показывает номер или место шлоки и короткий фрагмент канонического текста кириллицей, не дублируя название источника в каждой строке.
- [x] Действие редактирования на источнике открывает защищенную форму редактирования источника.
- [x] Форма редактирования источника позволяет менять название и описание источника, названия существующих частей и глав, а также добавлять новые части и главы.
- [x] Форма редактирования источника не позволяет менять код источника, тип структуры источника, коды существующих частей и глав или удалять существующие части и главы.
- [x] Действие редактирования на шлоке открывает защищенную форму редактирования шлоки.
- [x] Форма редактирования шлоки показывает источник, часть, главу и поле `Номер шлоки` как неизменяемую ссылку на шлоку.
- [x] Форма редактирования шлоки позволяет менять только пада и опциональный полный перевод шлоки; первая пада остается обязательной.
- [x] Редактирование шлоки показывает предупреждение, что изменение канонического текста затронет всех пользователей общей библиотеки.
- [x] Код источника, код шлоки и ссылка на шлоку остаются неизменяемыми после создания.
- [x] В административном UI и API текущего MVP нет физического удаления, публикации, скрытия или пакетного импорта.
- [x] Тесты покрывают доступ администратора, отказ обычному пользователю, защищенную admin-ручку списка, отображение всех источников включая пустые, отсутствие плейсхолдера для пустого источника, стабильную сортировку шлок, переходы к созданию и редактированию, ограничения редактирования источника, ограничения редактирования шлоки, обязательную первую паду при редактировании и отсутствие удаления/публикации/скрытия.
