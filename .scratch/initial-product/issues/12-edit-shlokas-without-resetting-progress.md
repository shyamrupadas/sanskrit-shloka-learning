# 12 — Редактирование шлок без сброса прогресса

**What to build:** После появления личных статусов шлок и истории повторений проверить и при необходимости доработать уже существующее административное редактирование каталога так, чтобы правки канонической шлоки обновляли текст для всех пользователей без сброса личного прогресса. Этот срез не добавляет новые экраны админки: он использует страницу админки, список и защищенные формы редактирования из среза админского каталога.

**Blocked by:** None — can start immediately

**Status:** awaiting-human-review
Accepted: 2026-07-14

- [x] Редактирование источника и шлоки по-прежнему открывается из существующей страницы админки.
- [x] Редактирование уже доступной в общей библиотеке шлоки по-прежнему показывает предупреждение, что изменение затронет всех пользователей.
- [x] Редактирование канонического содержимого шлоки обновляет отображение для всех пользователей без сброса личного статуса шлоки или истории повторений.
- [x] Редактирование источника шлоки не сбрасывает личные статусы шлок или историю повторений шлок из этого источника.
- [x] Ссылка на шлоку, код шлоки и код источника остаются неизменяемыми.
- [x] В административном UI и API MVP нет действия физического удаления.
- [x] Публикации и скрытия шлок в административном UI и API текущего MVP нет.
- [x] Тесты покрывают сохранение статусов `буду учить` и `повторяю` после правок, сохранение истории повторений после правок, обновление отображаемого текста для пользователей, предупреждение при редактировании доступной шлоки, неизменяемость ссылки на шлоку, отсутствие физического удаления и отсутствие действий публикации/скрытия.

## Результат

- Подтверждено, что редактирование источника и канонического содержимого
  шлоки обновляет общие данные по стабильному коду без замены самой шлоки,
  поэтому связанные личные статусы и история повторений остаются на месте.
- Добавлен API integration regression-сценарий для двух пользователей: статус
  `буду учить`, статус `повторяю`, время перехода и завершенная история
  повторения сохраняются после последовательного редактирования шлоки и ее
  источника, а оба пользователя получают обновленные текст, перевод и название.
- Существующие web-тесты подтверждают переходы из админки, предупреждение и
  неизменяемые поля ссылки; contract-тест подтверждает отсутствие DELETE,
  публикации и скрытия в API.
- Production-код и схема БД не менялись; новых миграций нет.
