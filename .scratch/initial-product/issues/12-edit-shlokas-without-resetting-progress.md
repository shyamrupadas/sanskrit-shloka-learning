# Редактирование шлок без сброса прогресса

Status: ready-for-agent

## Родитель

`docs/prd/0001-initial-product.md`

## Что сделать

После появления личных статусов шлок и истории повторений проверить и при необходимости доработать уже существующее административное редактирование каталога так, чтобы правки канонической шлоки обновляли текст для всех пользователей без сброса личного прогресса. Этот срез не добавляет новые экраны админки: он использует страницу админки, список и защищенные формы редактирования из среза админского каталога.

## Критерии приемки

- [ ] Редактирование источника и шлоки по-прежнему открывается из существующей страницы админки.
- [ ] Редактирование уже доступной в общей библиотеке шлоки по-прежнему показывает предупреждение, что изменение затронет всех пользователей.
- [ ] Редактирование канонического содержимого шлоки обновляет отображение для всех пользователей без сброса личного статуса шлоки или истории повторений.
- [ ] Редактирование источника шлоки не сбрасывает личные статусы шлок или историю повторений шлок из этого источника.
- [ ] Ссылка на шлоку, код шлоки и код источника остаются неизменяемыми.
- [ ] В административном UI и API MVP нет действия физического удаления.
- [ ] Публикации и скрытия шлок в административном UI и API текущего MVP нет.
- [ ] Тесты покрывают сохранение статусов `буду учить` и `повторяю` после правок, сохранение истории повторений после правок, обновление отображаемого текста для пользователей, предупреждение при редактировании доступной шлоки, неизменяемость ссылки на шлоку, отсутствие физического удаления и отсутствие действий публикации/скрытия.
