# Заучивание переводит шлоку в `повторяю`

Status: ready-for-human
Accepted: 2026-07-12

## Родитель

`.scratch/initial-product/spec.md`

## Что сделать

Сделать поток заучивания для шлоки в статусе `буду учить`. Пользователь должен открыть экран заучивания, отметить, что шлока еще не выучена, или отметить ее выученной, чтобы она перешла в `повторяю` с сохранением времени перехода для будущего повторения и серии дней.

## Критерии приемки

- [x] Действие `Учить` на карточке `Буду учить` открывает поток заучивания этой шлоки.
- [x] Экран заучивания показывает отображаемое название шлоки, текст шлоки кириллицей и ссылку на советы в `Обучение санскриту`.
- [x] `Не выучил` оставляет шлоку в `буду учить` и не создает подробную историю заучивания.
- [x] `Выучил` сразу переводит шлоку из `буду учить` в `повторяю`, сохраняет время перехода в `повторяю` и не позволяет вернуть шлоку в доступное состояние.
- [x] После `Выучил` пользователь видит состояние завершения с действиями `Выучить еще` и `На дашборд`.
- [x] `Выучить еще` открывает список выбора, если в `буду учить` осталось больше одной шлоки, открывает единственную оставшуюся шлоку, если она одна, и открывает таб `Все`, если шлок не осталось.
- [x] Тесты покрывают `Не выучил`, `Выучил`, сохранение времени перехода, отсутствие подробной истории неудачного заучивания, действия завершения, все ветки `Выучить еще` и инвариант, что шлоку в `повторяю` нельзя удалить из повторения.

## Принятые UI-решения и исключения

- Решение пользователя от 2026-07-12: действие `Не выучил` возвращает во
  вкладку `Буду учить` без изменения статуса и без записи истории.
- Решение пользователя от 2026-07-12: при нескольких оставшихся шлоках
  действию `Выучить еще` соответствует экран `Библиотека — буду учить`
  (`g0MoYL`); при отсутствии оставшихся шлок открывается вкладка `Все`.
- Обновленные экраны `QPXXW` и `IQcqK` являются focused-flow без нижней
  навигации; frontend повторяет это поведение.
- Design exception по решению пользователя от 2026-07-12: во вкладке
  `Буду учить` карточка показывает действия `Учить` и `Убрать`, хотя
  `g0MoYL` и reusable-компонент `Vzs9b` пока не содержат этот action set.
- Design exception по решению пользователя от 2026-07-12: пока отдельное
  Pencil-состояние загрузки не добавлено, frontend показывает skeleton из
  одного заголовка и четырех строк.
- Состояние ошибки заучивания осознанно отложено и не входит в текущую задачу.

## Результат

- Добавлен защищенный focused-route
  `/library/shlokas/$shlokaCode/learn` без нижней навигации, с обновленными
  состояниями заучивания и завершения из Pencil.
- Карточка во вкладке `Буду учить` получила отдельное действие `Учить` рядом
  с существующим действием `Убрать`.
- Добавлен публичный API-переход `complete-learning`: он атомарно переводит
  только шлоку в статусе `learning` в `reviewing`, сохраняет
  `reviewing_started_at`, поддерживает безопасный повтор запроса и не позволяет
  вернуть шлоку в доступное состояние.
- Ответ перехода содержит оставшиеся шлоки к заучиванию, поэтому frontend
  реализует все три ветки `Выучить еще` без дополнительного промежуточного
  состояния.
- Добавлена DB migration `0006_reviewing_started_at`; ее необходимо применить
  в каждом окружении.
- Contract, API/repository и web integration tests защищают публичный endpoint,
  время перехода, инвариант статуса, отсутствие mutation после `Не выучил`,
  completion-действия, loading skeleton и все ветки `Выучить еще`.
