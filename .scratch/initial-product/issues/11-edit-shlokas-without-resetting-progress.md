# Редактирование шлок без сброса прогресса

Status: ready-for-agent

## Родитель

`docs/prd/0001-initial-product.md`

## Что сделать

Завершить административное редактирование источников шлок и шлок без физического удаления и сброса прогресса. Правки канонической шлоки должны обновлять текст для всех пользователей, сохраняя личные статусы и историю повторений. Редактирование должно использовать защищенные формы и не вводить отдельный экран списка админки.

## Критерии приемки

- [ ] Администраторы могут редактировать существующие источники шлок, структуру источников, метаданные шлоки, номер шлоки, пада и опциональный полный перевод шлоки через защищенные формы.
- [ ] Редактирование уже доступной в общей библиотеке шлоки показывает предупреждение, что изменение затронет всех пользователей.
- [ ] Редактирование канонического содержимого шлоки обновляет отображение для всех пользователей без сброса личного статуса шлоки или истории повторений.
- [ ] Редактирование не вводит отдельный экран списка админки; существующий источник или шлока могут открываться для редактирования через защищенный маршрут или выбор в форме.
- [ ] В административном UI и API MVP нет действия физического удаления.
- [ ] Публикации и скрытия шлок в административном UI и API текущего MVP нет.
- [ ] Тесты покрывают предупреждение при редактировании доступной шлоки, редактирование номера шлоки и пад, отсутствие отдельного экрана списка админки, отсутствие физического удаления, отсутствие сброса прогресса после правок и отсутствие действий публикации/скрытия.
