# 02 — Администратор создает первый источник и первую шлоку

**What to build:** Дать администратору возможность создать источник шлоки и валидную шлоку в общей библиотеке через защищенные административные формы. Созданная шлока сразу доступна пользователям для добавления в личный список; отдельной публикации в текущем MVP нет. Этот срез должен заложить авторизацию администратора, защищенные административные формы источника и шлоки, правила валидации и хранение канонического содержимого шлоки. Точки входа из настроек связываются с этими формами в отдельном срезе настроек.

**Blocked by:** None — can start immediately

**Status:** awaiting-human-review
Accepted: 2026-06-24

- [x] Пользователь может иметь роль администратора, не теряя обычное пользовательское поведение и возможность иметь личный прогресс.
- [x] Административные формы и API доступны только пользователям с ролью администратора; обычные пользователи не имеют доступа даже по прямому URL/API-запросу.
- [x] Администратор может создать источник шлоки с кодом источника, названием, опциональным описанием и структурой: без глав, с главами или с частями и главами внутри частей.
- [x] Администратор может создать шлоку через выбор источника шлоки, выбор главы при необходимости, явное поле `Номер шлоки` внутри выбранного места источника, 1-4 пада и опциональный полный перевод шлоки.
- [x] Латинская транслитерация, пословная транслитерация, пословный перевод и деванагари не обязательны в текущем MVP.
- [x] Код источника уникален и неизменяем после создания; код шлоки уникален и неизменяем после создания; ссылка на шлоку уникальна.
- [x] Созданная шлока сразу появляется в API общей библиотеки шлок в стабильном порядке по источнику и номеру.
- [x] Срез не добавляет отдельный экран списка админки, публикацию, скрытие, физическое удаление или пакетный импорт.
- [x] Тесты покрывают доступ администратора, отказ обычному пользователю, обязательные метаданные источника и шлоки, структуры источника, поле `Номер шлоки`, обязательную первую паду, порядок пад, опциональный полный перевод, уникальные коды, неизменяемые коды, уникальную ссылку на шлоку и отсутствие обязательной латиницы/пословного перевода/деванагари.
