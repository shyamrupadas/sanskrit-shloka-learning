# 05 — Библиотека шлок и статус `буду учить`

**What to build:** Сделать пользовательскую библиотеку шлок с табами `Повторяю`, `Буду учить` и `Все`. Пользователь должен находить шлоки общей библиотеки, добавлять доступную шлоку в `буду учить`, убирать ее обратно в доступное состояние и видеть пустые состояния, объясняющие следующий шаг.

**Blocked by:** None — can start immediately

**Status:** awaiting-human-review
Accepted: 2026-07-13

- [ ] Маршрут библиотеки открывается с табом `Повторяю` по умолчанию.
- [ ] Таб `Все` показывает все шлоки общей библиотеки в стабильном порядке по источнику шлоки и номеру шлоки.
- [ ] Таб `Все` поддерживает поиск по отображаемому названию шлоки, источнику шлоки и номеру шлоки; бэкенд-поиск не обязателен, если фронтенд-поиска достаточно для MVP.
- [ ] Карточки шлок показывают отображаемое название шлоки, личный статус и действие для текущего таба, но не показывают паду шлоки или другой фрагмент канонического текста.
- [ ] Для источника с числовым названием главы отображаемое название записывается без запятой после источника и с точкой между главой и номером шлоки: `Бхагавад-гита 1.1`.
- [ ] Пользователь может добавить доступную шлоку из `Все` в `буду учить`; после этого она появляется в `Буду учить` и больше не выглядит доступной для этого пользователя.
- [ ] Пользователь может убрать шлоку из `буду учить`; она возвращается в доступное состояние для этого пользователя.
- [ ] Есть пустые состояния для `Повторяю` и `Буду учить`, а также состояние без результатов поиска для `Все`.
- [ ] Тесты покрывают таб по умолчанию, шлоки общей библиотеки в `Все`, сортировку по источнику и номеру, поиск, добавление в `буду учить`, удаление из `буду учить`, действия карточек по статусу, формат отображаемого названия, отсутствие пад шлоки в карточках и обязательные пустые состояния.
