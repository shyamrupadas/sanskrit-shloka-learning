# Поддержать кириллическую транслитерацию с диакритиками во frontend

Status: ready-for-agent

## Родитель

`.scratch/cyrillic-transliteration-diacritics/PRD.md` — все user stories 1–35.

## Что сделать

Реализовать единый end-to-end frontend-срез поддержки кириллической транслитерации с комбинируемыми диакритическими знаками. Уже сохранённый санскритский контент и относящиеся к нему названия должны использовать локальный утверждённый Noto Serif Italic в соответствии с Pencil, а формы редактирования, переводы и обычный UI должны сохранить текущую типографику.

В том же срезе устранить функциональные Unicode-проблемы: подсказка повторения и каталожное превью должны сокращать текст только по границам графемных кластеров, а библиотечный поиск — находить названия без учёта регистра и диакритик, не изменяя сохранённую или показываемую строку. Точное происхождение шрифта, checksum, Unicode fixtures, границы scope и тестовые seams определены в родительском PRD.

Backend, API-контракты, схема БД и модель транслитерации не меняются.

## Критерии приемки

- [ ] Утверждённый WOFF2-файл Noto Serif Italic из PRD добавлен в frontend как локальный ассет под семантическим именем; его SHA-256 совпадает с зафиксированным в PRD.
- [ ] В репозитории присутствует лицензионный notice Noto SIL Open Font License 1.1 и зафиксировано происхождение шрифта; runtime не обращается к внешнему font host или CDN.
- [ ] Кодовый семантический токен санскритского текста синхронизирован с Pencil `$font-sanskrit`: используется Noto Serif и italic-начертание без изменения согласованных размеров, weights, цветов и line-height.
- [ ] Санскритский стиль применяется к сохранённому каноническому тексту на странице шлоки, в заучивании и раскрытых состояниях повторения, а также к санскритским словам в пословном разборе.
- [ ] Тот же стиль применяется ко всему готовому display title шлоки и к его возможным составляющим — названию источника, части и главы — в карточках библиотеки/дашборда, на странице шлоки, в заучивании и повторении.
- [ ] В административном каталоге стиль применяется к уже сохранённым названиям источников, составным строкам местоположения/заголовка и превью канонического текста.
- [ ] Специальный стиль не применяется к `input`, `textarea`, `select`, другим контролам создания/редактирования, переводам, обычным UI-подписям и сообщению «Текст скрыт…»; глобальный UI-шрифт и `$font-transliteration` не меняются.
- [ ] Строки `дхр̣тара̄шт̣ра ува̄ча` и `Бха̄гавата-пура̄н̣а 1.1` отображаются с видимыми `U+0323` и `U+0304`, без clipping, наложений и поломки desktop/mobile layout на согласованных экранах.
- [ ] Один общий frontend-механизм сегментирует текст по расширенным графемным кластерам и используется всеми изменяемыми операциями сокращения; независимые feature-local реализации Unicode-правил не создаются.
- [ ] Первая подсказка по строке `дхр̣тара̄шт̣ра ува̄ча` показывает `дхр̣тара̄шт̣...`, то есть первую половину первой непустой строки с округлением вверх по графемам; вторая подсказка показывает полную первую строку без потери или перестановки code points.
- [ ] Превью канонического текста в административном каталоге сохраняет текущие лимит и политику многоточия, но никогда не разделяет базовую букву и принадлежащие ей комбинируемые знаки.
- [ ] Поиск по уже поддерживаемым полям display title, источника, части и главы не зависит от регистра и диакритик: `бхагавата-пурана`, вариант в другом регистре и точное написание с диакритиками находят `Бха̄гавата-пура̄н̣а 1.1`.
- [ ] Поисковая нормализация применяется только к matching: сохранённые данные и текст результата остаются побитно неизменными; поиск по полному каноническому тексту не добавляется.
- [ ] Автоматические проверки покрывают существующие seams потока повторения, библиотечного matching, форматтера каталожного превью и design-token contract; тесты не зависят от случайных CSS-классов или полных DOM snapshots.
- [ ] Выполнена ручная браузерная проверка репрезентативных пользовательских и административных состояний по Pencil references, включая специальный шрифт в сохранённом контенте и обычный UI-шрифт в форме редактирования и скрытом состоянии повторения.
- [ ] Проходят обязательные web-проверки: lint, typecheck, unit/integration tests и production build; новый Playwright-сценарий только ради этой фичи не требуется.
- [ ] Backend, API, существующие строковые значения и схема БД не меняются; новые DB migrations не требуются.
