# Оформить известные UI Contract Collision для недостающих Pencil-состояний

Status: ready-for-agent

## Родитель

`.scratch/frontend-design-system-sync/PRD.md`

## Что сделать

Подготовить проверяемые записи по уже известным расхождениям между функциональными требованиями и Pencil UI-контрактом, чтобы последующие UI-срезы не решали недостающие состояния молча. Срез должен зафиксировать, где дизайн отсутствует или недостаточно точен, какие есть варианты решения для владельца продукта и какие будущие frontend-issues должны остановиться без принятого решения.

## Критерии приемки

- [ ] Для раскрытого состояния пачки повторения после `Показать еще N` оформлен `UI Contract Collision`.
- [ ] Для раскрытого состояния списка шлок к заучиванию после `Показать все` оформлен `UI Contract Collision`.
- [ ] Для специфичных пустых состояний вкладок `Повторяю` и `Буду учить` оформлен `UI Contract Collision` или явно указано, что используется общий `Product / Empty State`.
- [ ] Каждая collision-запись содержит требование, Pencil reference, расхождение, варианты решения и рекомендацию агента.
- [ ] Карта Pencil-узлов или связанный task context показывает эти collision как известные блокеры для будущих UI-задач.
- [ ] Ни одно недостающее состояние не реализуется в коде в рамках этого среза.
- [ ] Последующие issue-тексты могут ссылаться на эти collision-записи как на проверяемый источник ограничения.
